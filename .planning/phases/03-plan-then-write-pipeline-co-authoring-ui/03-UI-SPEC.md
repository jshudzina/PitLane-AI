# Phase 3 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — hand-authored Svelte CSS (no shadcn; SvelteKit 5 app, not React) |
| Preset | not applicable |
| Component library | none — custom Svelte components only |
| Icon library | Lucide (npm: lucide-svelte) — MIT, tree-shakeable, consistent stroke style |
| Font | system-ui stack: `-apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, Oxygen, Ubuntu, sans-serif` (matches pitlane-web baseline) |

Source: `pitlane-web/src/pitlane_web/templates/base.html` establishes font and color baseline. pitlane-studio extends this design language.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline element gap |
| sm | 8px | Compact element spacing, chip/tag inner padding |
| md | 16px | Default element spacing, card inner padding, form field gap |
| lg | 24px | Section padding, panel inner padding, card outer margin |
| xl | 32px | Layout column gaps, major component spacing |
| 2xl | 48px | Page section breaks, approval gate visual breathing room |
| 3xl | 64px | Page-level top/bottom padding |

Exceptions:
- Touch targets for the Approve Outline button: minimum 44px height (accessibility)
- Five-act sidebar row height: 56px (data density — fits act label + one data point without scroll)
- Beat editor toolbar height: 40px (TipTap convention)

---

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 15px | 400 | 1.6 |
| Label | 13px | 400 | 1.4 |
| Heading | 20px | 600 | 1.2 |
| Display | 28px | 600 | 1.15 |

Notes:
- Body 15px (not 16px) reduces visual weight in a data-dense two-panel layout
- Label 13px at weight 400 used for: confidence badges, signal type chips, act labels, data anchor text — visual distinction from body is served by size (13px vs 15px)
- Heading 20px used for: angle card titles, beat section headers, panel titles
- Display 28px reserved for: page-level article title only
- Monospace exception: data rationale text in angle cards, data anchor text in outline beats, and key data points in the five-act sidebar use `'Menlo', 'Consolas', monospace` at 13px / weight 400 / line-height 1.5 — visual distinction from prose is served by font-family, not size

## Layout

### Overall Shell

Three-column layout at desktop (1280px+ viewport — desktop only, per REQUIREMENTS.md Out of Scope):

```
┌─────────────────────────────────────────────────────────────────┐
│  Header (48px): PitLane Studio | Race: [selector] | [Export]   │
├──────────────┬───────────────────────────────┬──────────────────┤
│              │                               │                  │
│  Left Panel  │     Main Content Area         │  Right Sidebar   │
│  (280px)     │     (fluid, min 480px)        │  (260px)         │
│              │                               │                  │
│  Phase-aware │  Stage 1: Angle Cards         │  Five-Act        │
│  outline     │  Stage 2: Outline Panel       │  Sidebar         │
│  (after      │  Stage 3: Beat Editor         │  (always         │
│  approval)   │                               │   visible)       │
│              │                               │                  │
└──────────────┴───────────────────────────────┴──────────────────┘
```

- Left panel: hidden during angle selection (Stage 1); visible from Stage 2 onward showing outline beats as navigation
- Right sidebar: always visible, showing five-act data (ACT-03)
- Main content area: transitions between Stage 1, 2, and 3 states
- Total minimum width: 1100px; below this, layout does not adapt (desktop-only tool)

### Stage Transitions

| Stage | Trigger | Left Panel | Main Content | Right Sidebar |
|-------|---------|-----------|--------------|---------------|
| 1 — Race + Angle Selection | Initial load | Hidden | Race selector + angle cards | Five-act sidebar (empty state: "Select a race") |
| 2 — Outline Review | Angle selected | Visible (outline nav) | Outline panel + Approve button | Five-act sidebar (populated) |
| 3 — Beat Editing | Outline approved | Visible (beat nav, each beat as link) | TipTap beat editor | Five-act sidebar (populated) |

Transitions use a CSS opacity + transform fade (150ms ease-out). No full-page navigation — single-page Svelte app.

## Component Contracts

### Race Selector (UI-01, source: CONTEXT.md Claude's Discretion)

Two chained `<select>` dropdowns: Year (2023–current) then Round (1–N for selected year). Round dropdown is disabled until year is selected. No searchable list — simple and functional per CONTEXT.md.

- Year dropdown width: 100px
- Round dropdown width: 200px (shows "Round N — Circuit Name")
- Placement: inline in the header bar, right of "PitLane Studio" wordmark
- Loading state: round dropdown shows "Loading..." and is disabled while fetching rounds
- Error state: inline error text below selector pair in `#cf4444`

### Angle Cards (UI-01)

Grid layout: 2 columns, gap 16px. Each card: 200px min-height, `#242424` background, `#2e2e2e` border, border-radius 8px, padding 16px.

Card anatomy (top to bottom):
1. Signal type chip — 13px label weight 400, background `#1a1a1a`, border `#2e2e2e`, padding 4px 8px, border-radius 4px — text shows signal type (e.g. "SurpriseScore", "ΔR̂")
2. Angle name — 20px heading weight 600, `#e0e0e0`, margin-top 8px
3. Confidence badge — inline, right of heading row: pill shape, 3 variants:
   - High (≥0.7): background `rgba(58, 122, 74, 0.3)`, text `#6acc8a`, label "HIGH"
   - Medium (0.4–0.69): background `rgba(160, 120, 40, 0.3)`, text `#ccaa44`, label "MED"
   - Low (<0.4): background `rgba(120, 60, 60, 0.3)`, text `#cc8888`, label "LOW"
4. Data rationale — 13px monospace weight 400, `#999999`, line-height 1.5, margin-top 8px, max 3 lines with `text-overflow: ellipsis` on overflow

Selected state: accent border `#e10600` (2px), box-shadow `0 0 0 1px #e10600`, background `rgba(225, 6, 0, 0.06)`.

Hover state: border `#444444`, background `#2a2a2a` (subtle lift, no red — reserve red for selected).

Selection behavior: single-select only (v1). Click selects; clicking again deselects. "Develop This Angle" button appears below the grid when one card is selected.

### "Develop This Angle" Button

- Label: "Develop This Angle" (primary CTA — see Copywriting Contract)
- Style: background `#e10600`, text `#ffffff`, weight 600, 15px, height 44px, border-radius 6px, padding 0 24px
- Placement: below angle card grid, right-aligned
- Disabled state: background `#333333`, text `#666666`, cursor not-allowed — shown when no card selected
- Loading state (while outline generates): spinner icon left of label, label changes to "Generating Outline..."

### Outline Panel (UI-03)

Displayed in main content area during Stage 2. Full-width within content area.

Panel header: "Review Your Outline" (20px heading), subtitle "Edit beat titles and data anchors before approving. You cannot change these after approval." (13px label, `#999999`).

Beat list: vertical stack, gap 12px. Each beat row:
- Beat number badge: circular, 24px, `#1a1a1a` background, `#999999` text, 13px weight 400
- Beat title: editable `<input>`, full-width minus badge, 15px body weight 400, `#e0e0e0`, background `#1a1a1a`, border `#2e2e2e`, border-radius 4px, padding 8px 12px; focus border `#e10600`
- Data anchors: editable `<textarea>`, 13px monospace weight 400, `#999999`, background `#141414`, border `#252525`, border-radius 4px, padding 8px 12px, min-height 56px; focus border `#e10600`
- Add/remove controls: "Add Beat" link-button (13px, `#999999`, plus icon) at bottom of list; remove icon (trash, 16px `#555555`, aria-label="Remove beat") at right of each beat row — red on hover `#cf4444`
- Beat count constraint: minimum 1 beat, maximum 8 beats; "Add Beat" button is disabled (greyed) at 8

Reorder: up/down arrow icon buttons (16px, `#555555`) on each beat row — not drag-and-drop (CM-02 is deferred v2).
- Up arrow: aria-label="Move beat up"
- Down arrow: aria-label="Move beat down"

### Approve Outline Button (PTW-02, UI-03)

This is the hard gate UI element. Must be the most visually prominent element in Stage 2.

- Label: "Approve Outline and Generate Prose" (see Copywriting Contract)
- Style: background `#e10600`, text `#ffffff`, weight 600, 15px, height 48px (above minimum), border-radius 6px, padding 0 32px
- Placement: below beat list, full-width container with 48px top margin (visual breathing room)
- Gate warning text above button: "Once approved, you cannot edit beats. Prose generation will begin immediately." — 13px, `#999999`, italic, margin-bottom 12px
- Loading state: spinner, label "Generating Beat 1 of N..."
- After approval: button replaced by a success banner — green `#3a7a4a` background, text "Outline approved. Generating prose..." in `#6acc8a`

### Five-Act Sidebar (ACT-03)

Always visible in right sidebar column (260px wide). Sticky positioning within the sidebar scroll container.

Sidebar header: "Five Acts" — 13px label weight 400, `#999999`, uppercase, letter-spacing 0.08em.

Each act: stacked vertically, divider line `#2e2e2e` between acts.

Act row anatomy:
- Act number + label: 13px weight 400, `#e0e0e0` — e.g. "Act 1 — Grid & Qualifying"
- Data sources: 13px, `#999999`, comma-separated command names — e.g. "session-info, qualifying-results"
- Key data point: 13px monospace weight 400, `#777777`, 2-line max, ellipsis on overflow — populated from `FiveActMapper.fetch_act_data()` result
- Empty state (before race selected): each act shows "—" for data sources and key data point in `#444444`

Act labels (from ACT_CONFIG):
- Act 1: "Grid & Qualifying"
- Act 2: "Lap 1 Chaos"
- Act 3: "Pit Window"
- Act 4: "Final Stint"
- Act 5: "Championship Implications"

### Beat Editor (UI-02, PTW-03, PTW-04)

TipTap 2.x block editor rendered in main content area during Stage 3. Full-width within content area.

Layout: vertical stack of beat blocks, each beat in its own contained section.

Beat section anatomy:
1. Beat header bar (40px, background `#1a1a1a`, border-bottom `#2e2e2e`):
   - Beat number + title (13px label weight 400, `#e0e0e0`)
   - TipTap toolbar (bold, italic, only — minimal toolbar; no heading levels in beat prose)
2. TipTap editor canvas (background `#0f0f0f`, padding 16px 20px, min-height 120px)
3. Beat footer bar (32px, background `#141414`, border-top `#252525`): word count label (13px, `#555555`)

Streaming state (while SSE tokens arriving for this beat):
- Beat section background pulses: `#1a2a3a` — CSS animation `pulse-stream 1.5s ease-in-out infinite` on background-color
- Beat border: `#2a5a8a`
- Streaming indicator in header bar: animated dot "Generating..." (13px, `#5a9acc`)

Completed beats: restore to default dark styling. No streaming indicator.

### Placeholder Hook Nodes (PTW-04, UI-02)

Implemented as TipTap custom inline node extensions (not marks) — survives `getJSON()` round-trips (source: CONTEXT.md Claude's Discretion decision).

Three node types:

| Node Type | Label | Background | Border | Text color |
|-----------|-------|------------|--------|------------|
| `placeholderQuote` | "JOURNALIST: Add quote" | `#1a3a2a` | `#2a6a4a` | `#5aaa7a` |
| `placeholderContext` | "JOURNALIST: Add context" | `#1a2a3a` | `#2a4a6a` | `#5a8aaa` |
| `placeholderCausal` | "JOURNALIST: Add causal reasoning" | `#2a2a1a` | `#5a4a2a` | `#aaaa5a` |

Visual treatment for all three:
- Display: `inline-block`
- Font: 13px monospace weight 400
- Padding: 2px 8px
- Border-radius: 3px
- Border: 1px solid (type-specific color above)
- Cursor: `default` (non-editable node — journalist replaces the whole node with their prose)
- Icon: 13px icon left of label — `quote-icon` / `info-circle` / `git-merge` from Lucide

In markdown export (`XPRT-01`), these render as:
- `placeholderQuote` → `[JOURNALIST: quote]`
- `placeholderContext` → `[JOURNALIST: context]`
- `placeholderCausal` → `[JOURNALIST: causal]`

### Left Panel Outline Navigator (Stage 2 onward)

280px left panel. Shows outline beats as a navigation list.

Panel header: "Outline" — 13px label weight 400, `#999999`, uppercase, letter-spacing 0.08em.

Each beat: 40px row, padding 0 16px, 13px weight 400, `#999999`. Active beat (editor scrolled to this beat): text `#e0e0e0`, left border 2px `#e10600`.

Status indicator right of label:
- No prose yet: dash `—` in `#444444`
- Streaming: spinning dot in `#5a9acc`
- Complete: checkmark icon in `#6acc8a`

### Copy Markdown Button (XPRT-01)

- Label: "Copy Markdown" (see Copywriting Contract)
- Style: background `#e10600`, text `#ffffff`, weight 600, 13px, height 36px, border-radius 6px, padding 0 16px
- Placement: header bar, far right
- Available at all stages (can export partial draft at any point)
- Active feedback: label changes to "Copied!" for 2 seconds, then reverts — no toast or modal needed
- Behavior: calls `navigator.clipboard.writeText(markdownString)` — no server round-trip needed for export

## Interaction States

| State | Description |
|-------|-------------|
| Initial | Race selector active, angle cards area shows "Select a race to surface story angles" |
| Race loading | Round dropdown disabled, spinner in dropdown |
| Angles loading | Angle card area shows skeleton cards (3 cards, pulsing `#1a1a1a` → `#242424`) |
| Data not ready | Full-width banner in main content area: orange-tinted warning, message from `DataNotReadyError` |
| Angle selected | One card in accent-selected state; "Develop This Angle" button enabled |
| Outline generating | "Develop This Angle" button in loading state |
| Outline ready | Stage 2 rendered; beat list populated from LLM response |
| Approval pending | Approve button visible, active, labeled "Approve Outline and Generate Prose" |
| Prose generating | Beat 1 in streaming state; left panel shows streaming indicator on beat 1 |
| All beats complete | All beats show checkmark; approve button replaced by completion message |
| Export | Copy Markdown button always present; "Copied!" feedback on click |
| Backend error | Error banner: destructive red, specific error message + "Try again" action |
| 409 gate violation | Not surfaced to user — UI prevents reaching this state; shown only in browser devtools |

## SSE Event Schema (Claude's Discretion — CONTEXT.md)

Full payload spec for each event type per beat stream (`GET /articles/{id}/beats/{beat_number}/stream`):

| Event | Payload fields | Frontend action |
|-------|---------------|-----------------|
| `beat_start` | `beat_number: int`, `beat_title: str`, `total_beats: int` | Render beat block in streaming state; update left panel indicator |
| `token` | `beat_number: int`, `token: str` | Append token text to beat's TipTap editor content |
| `beat_done` | `beat_number: int`, `prose: str` (full), `placeholder_markers: list[{type, offset}]` | Finalize beat content; insert placeholder hook nodes at offsets; exit streaming state; trigger next beat SSE connection |
| `error` | `beat_number: int`, `message: str`, `retryable: bool` | Show error banner on beat block; show "Retry Beat" button if `retryable: true` |
